Tutorials

The following Jupyter notebooks walk through each component of fewtrax, with all code cells pre-executed and output visible inline. They are rendered here via nbsphinx.

Note

All notebooks call jax.config.update("jax_enable_x64", True) at the top. This must be done before importing any fewtrax module.

Notebook descriptions

Notebook

Description

01 — Kerr geodesics

Computing Boyer-Lindquist frequencies \(\Omega_\phi, \Omega_\theta, \Omega_r\) and the separatrix \(p_{\rm sep}(a, e)\) for equatorial orbits.

02 — EMRI trajectory

Integrating the adiabatic inspiral ODE and visualising how \((p, e)\) and the orbital phases evolve toward the separatrix.

03 — Amplitude interpolation

Loading and evaluating the Teukolsky mode amplitude B-splines as a function of orbital parameters.

04 — Mode summation

Coherent harmonic mode summation; choosing which \((l,m,k,n)\) modes contribute more than a given fractional threshold.

05 — Full waveform

End-to-end waveform generation, time-domain and frequency-domain analysis, and comparison with the FEW reference implementation.

06 — JAX features

JIT compilation, automatic differentiation (jax.grad), and batched parameter sweeps with jax.vmap.

Running locally

pip install "fewtrax[dev]"   # installs jupyter and matplotlib
jupyter notebook examples/

Pre-executing notebooks for the docs

The docs build uses nbsphinx_execute = "never", meaning that notebooks must be committed with output already saved. To regenerate output:

# Execute all notebooks in-place (requires FEW data files)
jupyter nbconvert --to notebook --execute --inplace examples/*.ipynb

# Then rebuild the docs
cd docs && make html

This separates the (potentially slow, data-dependent) execution from the (fast, portable) HTML build. On ReadTheDocs the notebooks are therefore rendered from the committed output cells without requiring the FEW HDF5 data files to be present on the build server.

Standalone quickstart script

examples/quickstart.py is a self-contained script covering the complete workflow without Jupyter:

export FEW_DATA_DIR=/path/to/few/data
python examples/quickstart.py
