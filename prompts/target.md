# JAX EMRI waveforms

Create a JAX implementation of the EMRI waveform model in FastEMRIwaveforms. Use the same API as the existing implementation, but make optimal use of the vmap, jit and automatic differentiation features of jAX to achieve high performance. Use the KerrEccentricEquatorial waveform model. 

Context: the goal of this prject is to have a complete jax-suite of waveform, LISA detector response and data analysis tools fopr EMRIs. 

## Details
- Divide the waveform generation into several steps:
    1. a trajectory module that computes the evolution of the orbital parameters (a, p, e) as a function of time, using the equations of motion and the radiation reaction forces. To comply with the jit and vmap features, the adaptive time-stepper method will not work. Use methods implemented in diffrax library. Apply some tests and visualizations of the accuracy of these trajectories, as they will determine the effectiveness of the envisioned search procedure. 
    2. an amplitude module that computes the amplitudes for all the modes (l, m, n) as a function of the orbital parameters. This includes a string interpolation over the existing amplitude grids. This can be done once using the precomputed data and the massive spline coefficients may be loaded on the GPU, and then the interpolation can be done on the fly on the GPU. Pay special attention to the accuracy of the interpolation methods implemented on the GPU, as this will affect the accuracy of the waveform and its derivatives. Use at least a cubic spline to maintain c2 differentiability, which is important for the accuracy of the waveform and its derivatives.
    3. a summation module that sums all the harmonic modes to produce a time domain waveform
    4. a waveform module that combines the above steps to produce the final waveform, including the correct frame transformations to produce a waveform in the detetctor frame.
- The FEW framework is based on large datasets of precomputed fluxes and amplitudes, stored in hdf5 files. Construct a data loading module that loads this data on the GPU and makes it available for the waveform generation. Pay special attentioj to memory management and the accuracy of the interpolation methods implemented on the GPU. Use at least a  cubic spline to maintain c2 differentiability, which is important for the accuracy of the waveform and its derivatives.
- Add several utility functions:
    - A function to compute single harmonic tracks (l, m, n) in time-frequency domain, which can be used for visualization and debuggig. Additionally, we would like to use these tracks to construct a hierarchical multi-mode search algorithm. This idea exploits the structure between the different modes, which are integer multiples of the fundamental frequencies evolution. By searching for a single track using a semi-coherent STFT based method with a fresnel kernel (See paper by Rodrigo tenorio and davide gerosa, and the single harmonic search algorithm developed by Lorenzo Speri and Christian Chapman-Bird), we can identify a candidate EMRI track. Based on this identified track, we can then construct a template bank of waveforms that are consistent with the identified track, and perform a more detailed search using these templates. This hierarchical approach can significantly reduce the computational cost of the search, while still maintaining high sensitivity to EMRI signals. By filetering through the newly found tracks, we can solve for the most probable harmonic indices (l, m, n) and the corresponding orbital parameters (a, p, e) that are consistent with the identified track. This can be done using a combination of analytical approximations and numerical methods, such as the adam optimizer and differential evolution. By iteratively refining the search based on the identified tracks, we can efficiently explore the parameter space and identify potential EMRI candidates in the data. In a final stage, we perform a full bayesian inference study using the accurate fastEMRIWaveforms to reconstruct the posterior distribution of the EMRI parameters, and to perform model selection between different EMRI models.
    - A function to transform the waveform to the frequency domain, useful for parameter estimation and data analysis. 
    - A function to compute the waveform in time-frequency domain (wavelets and WDM basis), useful for data analysis. You can use the package pywavelet for this purpose, it has a jax implementation.
