    "- **Dynamic arguments** (numeric parameter values like `p0`, `e0`, `a`) never trigger recompilation"

    "## 6.2  vmap: Batching Over Parameter Space\n",
    "\n",
    "`jax.vmap` vectorises a function along a batch axis without writing explicit loops.  For EMRI parameter estimation, the key use cases are:\n",
    "\n",
    "1. **Evaluating waveforms at a grid of parameter points** (for likelihood surfaces)\n",
    "2. **Batching Fisher matrix columns** (computing all ∂h/∂θᵢ in one pass)\n",
    "3. **Parallel MCMC chains** (each chain evaluates the waveform at a different point)\n",
    "\n",
    "### How vmap works\n",
    "\n",
    "```python\n",
    "# Scalar function: scalar input → scalar output\n",
    "f(x)  # x is a float\n",
    "\n",
    "# Batched version: vector input → vector output\n",
    "jax.vmap(f)(xs)  # xs has shape (N,)\n",
    "```\n",
    "\n",
    "Under the hood, vmap vectorises the XLA computation — it does NOT loop over the batch dimension.  On GPU this gives near-linear speedup with batch size."

    "## 6.5  Fisher Matrix (Trajectory Phase)\n",
    "\n",
    "The **Fisher information matrix** sets the Cramér–Rao lower bound on parameter estimation uncertainty.  In the large-SNR limit:\n",
    "\n",
    "$$\n",
    "\\Gamma_{ij} = \\left( \\frac{\\partial h}{\\partial \\theta_i} \\,\\middle|\\, \\frac{\\partial h}{\\partial \\theta_j} \\right)\n",
    "= \\int_0^\\infty \\frac{\\tilde{h}_i^*(f)\\, \\tilde{h}_j(f)}{S_n(f)} \\, df\n",
    "$$\n",
    "\n",
    "For a phase-dominated signal (the EMRI case), the dominant contribution comes from the phase gradient:\n",
    "\n",
    "$$\n",
    "\\Gamma_{ij} \\approx \\text{SNR}^2 \\times \\left\\langle \\frac{\\partial \\Phi}{\\partial \\theta_i} \\frac{\\partial \\Phi}{\\partial \\theta_j} \\right\\rangle_t\n",
    "$$\n",
    "\n",
    "Here we compute the Jacobian $\\partial(\\Phi_\\phi, \\Phi_r)_{\\rm end} / \\partial(p_0, e_0, a)$ using `jax.jacobian`."
